# Historical Timeline

Интерактивное React-приложение с исторической шкалой времени. Пользователь может переключаться между тематическими периодами, смотреть диапазон лет и просматривать события внутри выбранного периода через свайпер.

## Что есть в проекте

- круговая навигация по периодам;
- анимированное отображение начального и конечного года;
- слайдер событий для активного периода;
- загрузка данных через `@tanstack/react-query` и `axios`;
- локальный mock API через `MSW` в development-режиме.

## Установка

```bash
npm install
```

## Запуск

Локальная разработка:

```bash
npm start
```

Приложение поднимется на `http://localhost:3000`.

В режиме разработки автоматически запускается `MSW`, поэтому запрос к `/api/timeline-periods` обслуживается из файла [`src/mocks/handlers.ts`](/d:/Work/historical-timeline/src/mocks/handlers.ts).

## Сборка

Production-сборка:

```bash
npm run build
```

Результат будет собран в директорию `dist/`.

## Архитектура

Структура проекта близка к FSD:

- `src/app` - точка входа, провайдеры, глобальные стили;
- `src/pages` - страницы приложения;
- `src/widgets` - крупные UI-блоки, включая основной таймлайн;
- `src/features` - сценарии работы с периодами и свайпером;
- `src/entities` - сущности периодов и событий;
- `src/shared` - общие UI-компоненты, конфиг и утилиты;
- `src/mocks` - mock API для локальной разработки.

Webpack-алиасы настроены для слоёв `@app`, `@pages`, `@widgets`, `@features`, `@entities` и `@shared`.

## Данные

Сейчас приложение работает на моковых данных с тематическими периодами:

- наука;
- литература;
- кино;
- музыка;
- технологии;
- искусство.

При подключении реального backend достаточно сохранить контракт ответа `GET /api/timeline-periods`.
